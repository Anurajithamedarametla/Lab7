MadLib Game











Addendum

Approach and Choices:
Server-Side Implementation: To handle form submission and generate the mad lib result, I utilized Express.js, a popular web framework for Node.js. The server listens for POST requests on the root URL ('/') and extracts the form data using body-parser middleware.
HTML and CSS Styling: The HTML structure is simple, containing a form with input fields for a noun, adjective, verb, adverb, and place. I also included a footer displaying the last modified date using JavaScript. CSS styles were applied to enhance the appearance of the form and result.
Result Page Styling: Upon form submission, a result page is generated dynamically using a template literal in the server's response. The result is styled using a separate CSS file to maintain consistency and improve readability.
Modifications Made:
HTML: Incorporated input fields for adverb and place to enhance the mad lib experience.
CSS: Adjusted font sizes, colors, and layout for better visual appeal and readability. Added hover effects to buttons and links for interactivity.
Server-Side Logic: Updated the server.js file to handle the additional form fields and generate the mad lib accordingly. Included comments for clarity and maintainability.
Conclusionbr> By combining HTML, CSS, and Express.js, I created an interactive mad lib game where users can fill in words to generate creative stories. The project demonstrates proficiency in web development fundamentals and server-side programming using Node.js.
